Basic Search Pane for Anatomy Browser

  • User Story 38 - Basic Anatomy Search Pane

Contents: Title | Test Requirements | Manual Tests | SQL Tests

Test Requirements

  1. Provide the user with a single search box and button in the Search Pane of the Anatomy Browser
  2. Query the EMAPA ontology (i.e. not EMAPS) for each user search
  3. Query EMAPA terms and synonyms (not ids or definitions)
  4. Perform contains search in same way as Anatomy Search on GXD QF
    1. Match to full word by prefix.
    2. Match all tokens of a multiple token query.
  5. Return a term just once even when the term and/or its synonyms are matched multiple times.
  6. Return results in Solr "best match" order
  7. Limit results to 500.
  8. Provide a count of results to the user. If over 500 display 500+.
  9. Display matching text in bold for applicable terms.
  10. For each result display the term and its Theiler Stage range
  11. Clicking on one of the result terms should update the Term Detail and Tree View portion of the page.
  12. This user story does not include:
    1. autocomplete
    2. wildcards
    3. stemming
  13. See mockup for headings, text, spacing, general layout guidelines
  14. Provide these features on all supported browsers - IE, Firefox, and Safari.
