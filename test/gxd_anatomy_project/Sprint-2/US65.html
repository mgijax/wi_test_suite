• User Story 65 - Batch update to stage ranges in OBO file

Contents: Title | Test Requirements | Manual Tests | SQL Tests

Test Requirements

  1. Accept a tab-delimited input file with updates to stage ranges to the EMAPA OBO file
  2. Use 3 columns of the input file identified by column heading â€“ ignore others.
  3. The 3 columns of data required are:
    1. "EMAPA id" (nnnnn)
    2. "starts_at (2)" Theiler Stage (TSnn)
    3. "ends_at (2)" Theiler Stage (TSnn)
  4. Valid to have either or both of the new Theiler stage values the same as what is in the original OBO file
  5. Verify that "starts_at" Theiler Stage is less than or equal to "ends_at" Theiler Stage.
  6. If a row does not have either or both of the Theiler Stage values - don't update that field in the OBO file
  7. Valid values for the Theiler Stage fields are 1-28.
  8. If EMAPA id not found in OBO - skip, don't update file
  9. Provide basic error message for each of the validation checks so the file can be corrected.
  10. No curator interface required (an SE will run this process)

Tests

Manual tests

  1. Verify an update file with no issues updates the OBO file as expected. The file contains only the 3 required fields.

    new test

    1. Update both stage values for a term to 1-28.
    2. Update both stage values for a term to 2-2.
    3. Update only the starts_at value - ends_at same as OBO.
    4. Update only the ends_at value - starts_at same as OBO.
    5. Update only the starts_at value - ends_at blank in update file
    6. Update only the ends_at value - starts_at blank in update file
    7. Include a row where the stage range is the same as the OBO file.
    8. Include a term twice in the update file. Expect last occurrence to be in the OBO file.
  2. Verify an update file with no issues updates the OBO file as expected. The file contains the 3 required fields -- plus fields to ignore.

    new test

    1. Update both stage values for a term to 1-28.
    2. Update both stage values for a term to 2-2.
    3. Update only the starts_at value - ends_at same as OBO.
    4. Update only the ends_at value - starts_at same as OBO.
    5. Update only the starts_at value - ends_at blank in update file
    6. Update only the ends_at value - starts_at blank in update file
    7. Include a row where the stage range is the same as the OBO file.
    8. Include a term twice in the update file. Expect last occurrence to be in the OBO file.
  3. Verify an update file with several issues plus good data updates the OBO file as expected.

    new test

    1. Include a row where starts_at is greater than ends_at. 2-1.
    2. Include a row where starts_at is 0.
    3. Include a row where ends_at is 29.
    4. Include a row for an EMAPA id that is not in the OBO file
    5. Include valid updates from test 2.
