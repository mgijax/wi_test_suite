Test Requirements

  1. Accept a tab-delimited input file with updates to stage ranges to the EMAPA OBO file
  2. Use 3 columns of the input file identified by column heading â€“ ignore others.
  3. The 3 columns of data required are:
    1. "EMAPA id" (EMAPA:nnnnn)
    2. "starts_at" Theiler Stage (numeric)
    3. "ends_at" Theiler Stage (numeric)
  4. Valid to have either or both of the new Theiler stage values the same as what is in the original OBO file
  5. Verify that "starts_at" Theiler Stage is less than or equal to "ends_at" Theiler Stage.
  6. If a row does not have either or both of the Theiler Stage values - don't update that field in the OBO file
  7. Valid values for the Theiler Stage fields are 1-28.
  8. If EMAPA id not found in OBO - skip, don't update file
  9. Provide basic error message for each of the validation checks so the file can be corrected.
  10. No curator interface required (an SE will run this process)
