Test Requirements

  1. Display EMAPS terms/ids that are part of the stage-specific Mouse Anatomy vocabulary that is derived from EMAPA
  2. Display information about an EMAPS term on the new EMAPS Term Detail page.
    1. Anatomy Term - display as found in OBO file
    2. Theiler Stage - display stage of the term
    3. EMAPS id - displays the generated EMAPS ID derived by adding a 2 digit TS at the end of an EMAPA id -- EMAPS:nnnnnSS (primary id only)
    4. Synonym(s) - display synonyms found in OBO file, if present, in comma-delimited list.
    5. Definition - display definition found in OBO file, if present. Length - same as other OBO Vocabs - NN characters.
    6. Parent(s) - display edge and term of each parent from OBO file
  3. Display page heading and sub-heading as defined by GXD - see PPT mockup.
  4. See PPT EMAPA mockup for LAF (static text, field order, page layout vision).
  5. Provide user a means to link to the other related pages for this term
    1. Use a pull down list that includes TS # and dpc values - see PPT mockup
    2. List of Theiler stages is limited to the valid range of stages for this term and each link to a EMAPS page
    3. Include an option for the user to link to the corresponding EMAPA page for this term
    4. Only allow a single selection
    5. Provide mouse over text with dpc info that matches what is done on the GXD QF
    6. Provide help text next to pull-down as defined by GXD.
    7. Theiler stages in pull-downs and pop-ups do not display a leading zero when a single digit.
  6. Provide user a link to browse Stage Descriptions at www.emouseatlas.org (bring up a new tab)
  7. Provide user a means to link to the EMAPS parent term or terms for the displayed EMAPS child term.
    1. List each parent on it's own line
    2. List is-a parents before part-of parents (no secondary order)
    3. Link to the EMAPS parent term
  8. Include meta data values for Search Engine Optimization (SEO) -- proposal in review.
    1. Title
    2. Description
    3. Keywords
    4. Robot - NOODP and NOYDIR
  9. Display should be useable in all supported Browsers (IE, Firefox, Safari)
