Anatomy search uses EMAPS - Differential QF

  • User Story 86 & 98 - Differential QF - structure & stage/structure ribbons

Contents: Title | Test Requirements | Manual Tests | SQL Tests

Test Requirements

  1. An auto-complete list is used for all four instances of the Anatomy field on this Query Form. It functions the same way as the A/C on the standard QF (US 83).
  2. Use the "selected" item from the Autocomplete list for each field to match EMAPS terms, perform comparisons, and return GXD results.
  3. Step 1 -- Match EMAPA terms/synonyms using the words of the selected item on the QF for each anatomy field to find a "bucket" of genes to use in comparison.
    1. Full word matching - no prefix matching, no stemming
    2. When more than 1 word is present in the "selected" item -- all those words must be present for a match to occur.
    3. Order of words not significant when matching
    4. Ignore special characters when matching terms/synonyms
    5. Include results for all children of every matched EMAPA term
    6. Continue the use of stop words (as is done now on Public) - and, from, of, or, the, their, to
    7. For the structure/stage ribbon -- include the stage selections, if any, to determine the "bucket" of genes for comparison.
  4. Step 2 -- Compare the 2 "buckets" of genes and return the appropriate list of genes per search rules (rules not changed by this story)
  5. GXD Summary results page should not be impacted by this user story (You Searched For, data, sort order, etc)
  6. Search should work properly in all supported browsers (IE, Firefox, Safari)
