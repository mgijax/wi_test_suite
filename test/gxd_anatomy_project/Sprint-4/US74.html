AD/EMAPS EI Module

  • User Story 74 - AD/EMAPS EI Module

Contents: Title | Test Requirements | Manual Tests | SQL Tests

Test Requirements

  1. Provide new EI module to edit the new EMAPS-AD mapping table.
    • After the initial load of the mappings - this will be the only way for curators to modify the mapping translations.
  2. Validation of fields is not required or desired - verification of the mappings will be done by a QC report.
  3. EMAPS terms may have multiple AD terms mapped to it -- each row of the Acc ID table represents an AD term
  4. Provide a Search function
    1. Fields with search enabled: EMAPS ID, EMAPS term, Acc ID (i.e. AD term)
    2. Fields without search: Structure, Stage, Created by/Modified by fields
    3. Accept a wildcard in all search fields
    4. To search by Acc ID the user must enter a value in the first row of the Acc ID table.
    5. Multiple field search??
  5. Provide a Clear function - clears all fields
  6. Provide a Delete function
    1. Deletes entry for an EMAPS id
    2. Warn user before executing the delete
  7. Provide an Add function
    1. Required fields: EMAPS id, ...
    2. Fields not allowed during an Add: ....
    3. Optional fields: .....
    4. If a user adds a mapping for an EMAPS id that is already in the table, but to a different AD term....
    5. If a user adds a mapping for an EMAPS id/AD term combo that is already in the table...
    6. If a user adds a mapping for an AD term that is already in the table, but mapped to another EMAPS id....
    7. Allow a user to map an EMAPS id to 1 or more AD terms
    8. Validation: a user may enter any type of text in the ID fields - no validation is performed.
  8. Provide a Modify function
    1. Fields that can be modified directly: EMAPS id, ...
    2. Fields that cannot be modified: ....
    3. Use the "Add row" function to map a new AD term to an EMAPS id
    4. Use the "Delete row" function to remove an AD term mapping
    5. Use the Delete row and Add row functions to change the mapping from one AD term to another.
    6. What happens if I delete all rows of AD term mappings - is it allowed? If so, is it treated like a Delete?
  9. Use traditional EI interfaces; defaults; search limits; etc

Tests

Manual tests

  1. Search by EMAPS id - exact match, no match, wildcards - use Clear function between queries.

    new test

    1. Search for a valid EMAPS id that has a 1:1 mapping - EMAPS:1603904. Expect match with mapping.
    2. Search for a valid EMAPS id that has no mapping - EMAPS:nnnnnnn. Expect no match.
    3. Search for a valid EMAPS id that has a 1:2 mapping - EMAPS:3153321. Expect match with mapping.
    4. Search for an EMAPS id that is invalid and not in the table - EMAPS:1111100. Expect no match.
    5. Search for an EMAPS id that is in the table using a wildcard - %16058%. Expect matches to visceral endoderm.
  2. Search by EMAPS term - exact match, no match, wildcards - use Clear function between queries.

    new test

    1. Search for a valid EMAPS term that has a 1:1 mapping - embryo. Expect match with mapping.
    2. Search for a valid EMAPS term that has no mapping - xxxxx Expect no match.
    3. Search for a valid EMAPS term that has a 1:2 mapping - renal proximal straight tubule Expect match with mapping.
    4. Search for an EMAPS term that is invalid and not in the table - football Expect no match.
    5. Search for an EMAPS term that is in the table using a wildcard - %renal artery%. Expect matches to several terms
  3. Search by AD term id - exact match, no match, wildcards - use Clear function between queries.

    new test

    1. Search for a valid AD term id that has a 1:1 mapping - MGI:nnnnn. Expect match with mapping.
    2. Search for a valid AD term id that has no mapping - MGI:4855499. Expect no match.
    3. Search for a valid AD term id that is part of a 1:2 mapping - MGI:5004819. Expect match with mapping.
    4. Search for an AD term id that is invalid and not in the table - MGI:11111. Expect no match.
    5. Search for an AD term id that is in the table using a wildcard - MGI:48509%. Expect several matches.
  4. Search by multiple fields

    new test

    1. Search for a valid EMAPS id and AD term id that has a mapping - EMAPS:1626721 and MGI:4864448. Expect match with mapping.
    2. Search for a valid EMAPS term and AD term with wildcards - %renal% and MGI:nnnnn. Expect match.
  5. Attempt to search by fields not enabled for searching

    new test

    1. Search a structure; brain. Expect search to ignore entered value - return first 2000 records
    2. Search for a stage; 28. Expect search to ignore entered value - return first 2000 records.
    3. Search for each of the created by/mod by fields. Expect search to ignore entered value - return first 2000 records.
    4. Search for a field that is enabled for searching along with a field that is not. Expect field not enabled to be ignored.
  6. Use the Add function to enter a mapping for an EMAPS id

    new test

    1. Add a 1:1 entry using valid EMAPS/AD term ids that do not currently have a mapping.
    2. Add a 1:2 entry using valid EMAPS/AD term ids that do not currently have a mapping.
    3. Add a 1:2 entry using all invalid ids that do not have a mapping.
    4. Add a 1:3 entry using valid EMAPS/AD term ids that do not currently have a mapping
    5. Add a 1:1 entry using valid EMAPS/AD term ids where the EMAPS id already has a mapping.
    6. Add a 1:1 entry using valid EMAPS/AD term ids where the AD term id is already part of a mapping.
    7. Add a 1:1 entry using valid EMAPS/AD term ids where both already are mapped to each other.
    8. Add a 1:1 entry using valid EMAPS/AD term ids where both already are mapped, but not to each other.
  7. Use the Modify function to change existing mapping entries

    new test

    1. Modify an existing 1:1 entry - change the EMAPS id.
    2. Modify an existing 1:2 entry - change the EMAPS id.
    3. Modify an existing 1:1 entry - change the AD term id (must delete row/add row)
    4. Modify an existing 1:2 entry - change one of the AD term ids (must delete row/add row)
    5. Modify an existing 1:2 entry - change both of the AD term ids (delete both rows/add 2 rows)
    6. Attempt to modify term names and stages. Expect this not to be allowed.
    7. Modify an existing 1:1 entry - delete the AD term id and don't replace it. Expect: ?
  8. Delete the mapping for an EMAPS id

    new test

    1. Delete an EMAPS id mapping record that was entered via the load. EMAPS:nnnnn. Expect it to be deleted.
    2. Delete an EMAPS id mapping record that was entered via the EI. EMAPS:jilltest. Expect it to be deleted.
