Move EMAPA/EMAPS from back-end to front-end

  • User Story 15 - Move EMAPA/EMAPS from back-end to front-end

Contents: Title | Test Requirements | Manual Tests | SQL Tests

Test Requirements

  1. New Anatomy Browser works as expected with the full set of EMAPA and EMAPS vocabularies
    1. Term Detail works per requirements - US 11 & US 12
    2. Tree View works per requirements - US 38
    3. Search Pane works per requirements - US 77 & US 78
  2. Support search with special characters present in terms and synonyms
    1. comma, hyphen, apostrophe
  3. Support search of single character tokens in synonyms and terms
  4. Handle tree view display when the tree exceeds the viewable window
    1. Do not wrap terms - use horizontal scroll bar
    2. Move selected term to the window automatically
    3. Provide user with a vertical scroll bar when tree exceeds viewing window
  5. Anatomy Browser works in all supported browsers - IE, Safari, Firefox
