Add Feature Type filter

  • User Story 128 - Add Feature Type filter to HMDC

Test Requirements

  1. Filter mouse markers by Feature Type in the HMDC following the general paradigm of the GXD filters.
  2. Apply filter to all HMDC Tab results
    • Grid: keep matching mouse marker rows; keep human homologs of those markers
    • Genes: keep matching mouse marker rows; keep human homologs of those markers (continue to just display the direct matches in this tab, right?)
    • Diseases: keep diseases that are returned due to annotations to genes that match the filters
  3. Filter by Feature Type will remove all human genes that have no mouse homologs (i.e. a row with no mouse marker will never match any Feature Types).
  4. Move and re-label the current row/col 'remove' button to "Remove row/column filters".
  5. Add new filter button
    • Button label= Feature Type
    • Title of filter pop-up= Filter Mouse Genes by Feature Type
    • Display= dynamic list of FTs that are present in the mouse markers in the returned grid
    • Order= list in alpha order (case insensitive)
  6. Display a "remove" button for each filter type selected.
    • Will there be a 'Remove all filters' button?

Tests

Manual tests

  1. Verify row/column filter display and function

    new test

    1. Use r/c filter to select 1 row. Expect: 1 row displayed; remove button with new text/new location.
    2. Use r/c filter to select 1 column. Expect: 1 col displayed; remove button as above
    3. Use r/c filter to select mult rows/cols. Expect: selections to display; remove button to display.
    4. Use r/c remove button to unselect above filters. Expect: return to full grid.
  2. Verify initial list of Feature Types in filter pop-up

    new test

    1. Run query that returns markers with 1 feature type; XXX. Expect: 1 FT in filter.
    2. Run query that returns many feature types; XXX. Expect: all FTs in list in alpha order.
    3. Run query that returns typical number of feature types; XXX. Expect: all FTs in alpha order.
    4. Run query that returns no results; football. Expect: display message "No values in results to filter."
  3. Verify dynamic update of Feature Types in filter pop-up

    new test

    1. Select all FTs in filter. Expect: all FTs listed with checked box when re-opening filter.
    2. Select 1 FT in filter. Expect: only that FT listed when re-opening filter.
    3. Select partial set of FTs in filter. Expect: all selected FTs listed when re-opening filter.
  4. Verify results when filtering by Feature Type

    new test

    1. Run query that returns many FTs - select all via filter. Expect: filtered results the same as original results.
    2. Run query that returns 1 FT - select all via filter. Expect: Filtered results the same as original results.
    3. Run query that returns many FTs - select 1 via filter. Expect: Filtered results to include selected FT.
    4. Run query that returns many FTs - select 2-3 via filter. Expect: Filtered results to include selected FTs.
  5. Verify filter by Feature Type and human genes

    new test

    1. Run query that returns human genes w/no mouse homolog. Filter by FT. Expect: Those human genes no longer returned.
    2. Run query that returns human genes w/ mouse homolog - direct annotation. Filter by FT that matches homolog. Expect: Those human genes remain in results.
    3. Run query that returns human genes w/ mouse homolog - direct annotation. Filter by FT that does not match homolog. Expect: Those human genes to be removed from results.
    4. Run query that returns human genes w/ mouse homolog - indirect annotation. Filter by FT that matches homolog. Expect: Human gene remains in results.
    5. Run query that returns human genes w/ mouse homolog - indirect annotation. Filter by FT that does not match homolog. Expect: Those human genes to be removed from results.
  6. Verify filter by Feature Type and n-m homology

    new test

    1. Run query that returns a row with multiple mouse genes. Filter by FT that matches genes. Expect: mouse genes; human homologs to remain in results
    2. Run query that returns a row with multiple mouse genes. Filter by FT that does not match genes. Expect: mouse genes; human homologs no longer returned.
    3. Run query that returns a row with multiple human homologs. Filter by FT that matches genes. Expect: mouse genes; human homologs to remain in results.
    4. Run query that returns a row with multiple human homologs. Filter by FT that does not match genes. Expect: mouse genes; human homologs to remain in results.
    5. Any case where mouse homologs have different feature types? This would be an add'l test.
  7. Verify filter by Feature Type and text-matched diseases

    new test

    1. Run query that returns a disease via direct annotation and text match. Filter out direct annotation. Expect: disease to remain in disease tab.
    2. Run query that returns diseases via text match -- no genes; empty filter. Expect: display message "No values in results to filter."
  8. Verify remove filter buttons display in "filter area" and function

    new test

    1. Select 1 FT from the filter. Expect: 1 remove button to be displayed and click to remove to work.
    2. Select a long FT from the filter. Expect: remove button display to accommodate long text and click to remove to work.
    3. Run a query that returns a large number of FTs - select them all via filter. Expect: display to accommodate many buttons and clicks to remove each of them to work.
  9. Verify display in all supported browsers (Mac-FF, Safari, Chrome; PC-FF, Chrome)

    new test
