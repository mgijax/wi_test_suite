Add filters to the Interaction Explorer (simple fields)

  • User Story 84 & 85 - Add filters to the Interaction Explorer

Contents: Title | Test Requirements | Manual Tests | SQL Tests

Test Requirements

  1. Add filter options to the Interaction Explorer to provide the user a way to narrow down results.
    1. Interaction: list of unique terms present in results (currently just 1 term)
    2. Validation: list of unique values for this field present in results (currently: validated & predicted)
    3. Data Source: list of unique values for this field present in results (currently: 2 sources)
  2. All the filter options are present at the top of the page - and should filter the Table View only (filter of Graph different story)
    • See the GXD or Reference pages for style of filter buttons
    • Location of filter buttons - at the discretion of the SE (Jim/Wendy is this okay?)
  3. Filters are limited to the items present in the initial "bucket" of data and continually change as filters are added/removed
    1. When at least 1 item is checked for a filter and executed - the filter list will only include the checked items
    2. When all previously checked items are unchecked for a filter and executed - the original filter list will return
    3. Filtering items in 1 field may reduce the items to filter by in another.
  4. Users may select 1 or more items for each filter available.
  5. User may un-select 1 or more items for each filter available.
    • Provide two options: button at top of page or un-check of box within Filter
  6. Provide users a list of filters selected in the heading of the Explorer page as part of each filter event.
    • Each item is displayed as a button and also acts as an "undo" for the user.
  7. Provide users the option to remove all filters in 1 step.
  8. Provide users with updated pagination counts after each filter event
  9. Do we need to display original counts of summary (as the Reference Summary does) or not (as the GXD Summary does)?
  10. Browser controls
    1. Page refresh/reload -- removes all filters and returns to original "bucket" and order of results
    2. History -- forward/back buttons are "filter-aware" as implemented on GXD summary.
  11. Filters should work properly in all supported browsers (Mac: Firefox, Chrome, Safari; PC: Firefox, Chrome)
