Test Requirements

  1. Update load software to recognize & load the relationship category - "interacts_with"
    • Continue to limit the load to just the "Add" action
  2. Load a relatively small input file from Wendy (60K rows)
  3. Accept & load a file that contains both interacts_with and cluster relationships
  4. Update load software to recognize & load the following input columns that may be present for any category (not just interacts_with)
    1. Property fields - optional
    2. Notes - optional (255 character limit?)
  5. Add a sanity check regarding the location of the Property field columns in the input file ???????
    1. The Property fields must follow the Notes column (col 13) if they are present in the input file.
    2. Once a non-Property field column is found the remaining columns are ignored including a misplaced Property column.
    3. Add the output for the new Sanity checks to the existing Sanity check report file - sanity.rpt
  6. Add a QC check to validate the Property fields' column headings.
    1. Required prefix = Property:
    2. The text that follows the prefix must be present in the "Relationship Properties" controlled vocabulary
    3. There are no Property fields that are required to be present in the input file.
    4. Display line # and invalid column heading for each occurrence of this error.
    5. If this error is found during the load process the load will abort.
  7. Add a QC check to validate the data in the Property:score column (if the column is present)
    1. Valid values for score = blank, an integer, or a float
    2. plus (+) or minus (-) are allowed, but not required
    3. Display line # and invalid score for each occurrence of this error.
    4. If this error is found during the load process the load will abort.
  8. Add the output for the new QC checks to the existing QC report file (qc.rpt) with appropriate sub-headings
  9. Propose that we do not implement the following requirement that is documented in the wiki per curator feedback
    • Verify that each property column (if any exist) has at least one row with data
