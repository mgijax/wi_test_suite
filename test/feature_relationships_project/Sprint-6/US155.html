Test Requirements

  1. Score values will be scaled from 0-1 by the curator(s) that generate the input file -- this will enable us to provide a simple filter for Score.
  2. Provide user ability to filter by score using a slider: "Filter Predicted Interactions by Score".
  3. The score minimum and maximum values displayed in the filter are inclusive; i.e. scores with those values are included in the results.
  4. Score filter applies to any relationship with a score value (typically just Predicted) -- are we limiting it to Predicted in the s/w?
  5. In addition to returning relationships that are within the specified score filter range -- also return all relationships that have no score values.
  6. Display the minimum and maximum score present in the current set of data.
    1. minimum and/or maximum values updated after any filter event (score or any other filter)
  7. Display a single undo score button displaying ">=n.nnn" -- update the value upon further score filtering
  8. Include removal of score filtering in the "Remove all filters" button
