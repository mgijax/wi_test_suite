Allele Detail - Mutation Involves changes

  • User Story 119 - Implement allele detail changes for mutation involves

Contents: Title | Test Requirements | Manual Tests | SQL Tests

Test Requirements

  1. Add teaser of Mutation Involves relationships in the Mutation Description ribbon
  2. Display the total number of "related" markers
  3. Display up to 3 markers as a teaser for the user -- if more than 3 markers are related include an ellipsis.
  4. Link markers in the teaser list to their Marker Detail page.
  5. Order the markers in the teaser list in location order.
  6. Display a "View All" button that when clicked displays the new pop-up page of related markers.
  7. Display consistently in all supported browsers (PC - Chrome, Firefox; Mac - Chrome, Firefox, Safari)

Tests

Manual tests

  1. Verify placement and text of Mutation Involves in the Mutation Description ribbon.new test

    1. View mutation with a single mutation type and a short note; MGI:nnnnn. Expect: Involves text/teaser displayed.
    2. View mutation with multiple mutation types and a long note; MGI:nnnnn. Expect Involves text/teaser displayed.
    3. Possible to have a mutation without a note or mutation types?
  2. Verify related marker count.

    new test

    1. View mutation with markers that each have a single relationship; MGI:nnnnn. Expect: count=nn.
    2. View mutation with markers that have more than 1 relationship to the mutation; MGI:nnnn. Expect: count = nn.
  3. Verify marker teaser list ellipsis inclusion and links.

    new test

    1. View mutation with less than 3 related markers; MGI:nnnnn. Expect: Markers displayed with no ellipsis.
    2. View mutation with 3 related markers; MGI:nnnnn. Expect: Markers displayed with no ellipsis.
    3. View mutation with 4 related markers; MGI:nnnnn. Expect: Markers displayed with ellipsis.
    4. View mutation with many related markers; MGI:nnnnn. Expect: Markers displayed with ellipsis.
  4. Verify sort order of teaser list.

    new test

    1. View mutation with single relationships to markers; MGI:nnnnn. Expect: teaser markers in location order.
    2. View mutation with multiple relationships to a single marker; MGI:nnnnn. Expect: teaser markers in location order.
  5. Verify display/navigation changes on all supported browsers (PC: Firefox, Chrome; Mac: Firefox, Chrome, Safari)

    new test

    1. Display and action of "View All" button.
    2. Open/close action of note field in Mutation description ribbon.
    3. Links in teaser list
