QC Checks

  • User Story #3 - Initial QC checks for the coordinates & associated data in the Input File

Test Requirements

  1. Fail the QC check process for files with any of these errors and inform the curator of the error condition
    1. Header record does not include the "Build=" parameter
    2. duplicate lines are found in the input file
    3. duplicate MGI IDs are found in the input file
    4. lines other than header record have fewer than 7 columns
    5. required columns are blank
    6. first column begins with "MGI:" (except header record)
  2. Accept and correct the following conditions in an input file during the QC check process.
    1. Columns 8-n may be used by curators with no impact to QC or Load
    2. Blank lines within the input file are ignored
    3. Ctrl-M characters are ignored (unprintable control character sometimes inserted at the end of a line when moving a file from Windows to Unix/Linux)
  3. Report the following errors to the curator during the QC check. Reject on a line-by-line basis for the following conditions.
    1. MGI ID does not exist in the database
    2. MGI ID is not a mouse marker
    3. MGI ID status is not official or interim
    4. MGI Marker ID is not a primary ID (i.e. secondary IDs not processed)
    5. start coordinate is greater than the end coordinate
    6. chromosome is not a valid chromosome (i.e. 1-19, X, Y, XY are valid)
    7. coordinates are not integer values
    8. strand is not +, -, or blank
  4. Report the following information to the curator during the QC check. No data is rejected due to these conditions.
    1. list the marker when the chromosome in the input file does not match the chromosome for the marker in the database
    2. list the new 'source/display' pairs found in the input file that are not already in database
    3. display the Build text
      • from input file
      • all distinct build text for coordinates in database

Manual tests

    Sanity Checks

  1. Abort processing for input file when the header record does not include the Build parameter

    new test

    1. QC check an input file that does not have a header record. This should fail the QC check and report it to the curator.
    2. QC check an input file that has a header record with parameters, but does not include a Build parameter. This should fail the QC check and report it to the curator.
    3. QC check an input file that has a header record with multiple parameters. The Build parameter is included at the end of the list. This should pass the QC.
  2. Abort processing for input file when duplicate lines are found.

    new test

    1. QC check an input file that has 2 consecutive duplicate lines. This should fail the QC check and report it to the curator.
    2. QC check an input file that has 3 non-consecutive duplicate lines. This should fail the QC check and report it to the curator.
  3. Abort processing for input file when duplicate MGI IDs are found.

    new test

    1. QC check an input file that has 2 matching IDs. This should fail the QC check and report it to the curator.
    2. QC check an input file that has 3 matching IDs. This should fail the QC check and report it to the curator.
  4. Abort processing for input file when fewer than 7 columns are found.

    new test

    1. QC check an input file that has columns 1, 2, & 3 only. This should fail the QC check and report it to the curator.
    2. QC check an input file that has columns 1 and 6. This should fail the QC check and report it to the curator.
    3. QC check an input file that has columns 1, 6, 7 & 8 only. This should pass the QC check.
    4. QC check an input file that has columns 1,2,3,4,5,6,7,8. This should pass the QC check.
  5. Abort processing for an input file containing blanks in required columns.

    new test

    1. QC check an input file that has blanks in a single column other than strand. This should fail the QC check and report it to the curator.
    2. QC check an input file that has blanks in multiple columns other than strand. This should fail the QC check and report it to the curator.
    3. QC check an input file that has a blank strand column. This should pass the QC check.
    4. QC check an input file that has a valid delete record. Values in columns 1, 6, 7 and all others blank. This should pass the QC check.
    5. QC check an input file that has a valid delete/reload record. Values in columns 1,2,3,4,5,6,7 and all others blank. This should pass the QC check.
  6. Abort processing for an input file when the first field in lines 2-n do not begin with "MGI:".

    new test

    1. QC check an input file that has a line that begins "MGIx:". This should fail the QC check and report it to the curator.
    2. QC check an input file that has a line that begins "MGI9". This should fail the QC check and report it to the curator.
    3. QC check an input file that has a line that begins "mgi:". This should pass the QC check.

    7. Accept & Correct

  7. Allow curators to add notes in columns 8-n in the input file

    new test

    1. QC check an input file that has curator information in column 8 of the first line. This should pass the QC check.
    2. QC check an input file that has curator information in column 8 for several lines. This should pass the QC check.
  8. Accept input files with blank lines.

    new test

    1. QC check an input file with a single blank line. This should pass the QC check.
    2. QC check an input file with consecutive blank lines. This should pass the QC check.
  9. Accept input files with ctrl-M characters.

    new test

    1. QC check an input file with a single ctrl-M character. This should pass the QC check.
    2. QC check an input file with consecutive ctrl-M characters. This should pass the QC check.

    10. Line Rejects

  10. Report to the curator when an MGI ID is not in the database. Reject that line.

    new test

    1. QC check an input file with an invalid MGI ID. Report discrepancy to curator and reject the line.
  11. Report to the curator when an MGI ID is not for a mouse marker. Reject that line.

    new test

    1. QC check an input file that includes an MGI ID for an allele. Report discrepancy to curator and reject the line.
    2. QC check an input file that includes an MGI ID for a non-mouse marker. Report discrepancy to curator and reject the line.
  12. Report to the curator when an MGI ID has a status other than Official or Interim. Reject that line

    new test

    1. QC check an input file that includes an MGI ID for a withdrawn marker. Report discrepancy to curator and reject the line.
    2. QC check an input file that includes an MGI ID for an Interim marker. This passes the QC check.
  13. Report to the curator when a secondary MGI ID is included in the file. Reject the line.

    new test

    1. QC check an input file with a secondary MGI ID for a marker. Report discrepancy to curator and reject the line.
  14. Report to the curator when the start coordinate is greater than the end coordinate. Reject the line.

    new test

    1. QC check an input file with a start coordinate greater than the end coordinate. Report discrepancy to curator and reject the line.
    2. QC check an input file with a start coordinate equal to the end coordinate. This passes the QC check.
  15. Report to the curator when the chromosome is not valid. Reject the line.

    new test

    1. QC check an input file with a chromosome of 20. Report discrepancy to curator and reject the line.
    2. QC check an input file with a chromosome of UN. Report discrepancy to curator and reject the line.
    3. QC check an input file with a lower case x. This passes the QC check.
  16. Report to the curator when the coordinates are not integer. Reject the line.

    new test

    1. QC check an input file with one of the coordinates including an alpha character. Report discrepancy to curator and reject the line.
    2. QC check an input file with both of the coordinates including an alpha character. Report discrepancy to curator and reject the line.
    3. QC check an input file with a coordinate with a decimal place. Report discrepancy to curator and reject the line.
  17. Report to the curator when the strand is something other than +, -, or blank. Reject the line.

    new test

    1. QC check an input file with an invalid strand. Report discrepancy to curator and reject the line.

    18. Report & Load

  18. List marker when chromosome in the input file does not match the chromosome in the database. Line accepted.

    new test

    1. Chromosomes don't match -- marker has chr and coordinates in database.
    2. Chromosomes don't match -- marker has only chr in database
    3. Chromosomes don't match -- marker has UN in database
  19. List new source/display pairs present in the Input File

    new test

    1. Input file has a single occurence of a new source/display pair
    2. Input file has multiple occurences of a new source/display pair
    3. Input file has multiple new source/display pairs
  20. List build text from Input File and all unique build texts in the database

    new test

    1. Input file has build text that is currently in the database
    2. Input file has build text that is new
