QC Checks

  • User Story #3 - Initial QC checks for the coordinates in the Input File

Test Requirements

  1. Abort processing for files with any of these errors and inform the curator of the error condition
    1. Header record does not include the "Build=" parameter
    2. duplicate lines are found in the input file
    3. duplicate MGI IDs are found in the input file
    4. lines other than header record have fewer than 7 columns
    5. required columns are blank
    6. first column for other than the header record begins "MGI:"
  2. Columns 8-n may be used by curators with no impact to QC or Load
  3. Blank lines within the input file are ignored
  4. Ctrl-M characters are ignored (unprintable control character sometimes inserted at the end of a line when moving a file from Windows to Unix/Linux)
  5. Double quotes within the input file are ignored
  6. Skip processing for invalid markers in the input file and create a report for the curator
    1. MGI ID does not exist in the database
    2. MGI ID is not a mouse marker
    3. MGI ID status is nto official or interim
  7. Skip processing for any MGI Marker ID that is not a primary ID (i.e. secondard IDs not processed)
  8. Skip processing for any marker where the start coordinate is greater than the end coordinate.
  9. Skip processing if the chromosome is not a valid chromosome (UN is not valid)
  10. Skip processing if the coordinates are not numeric values
  11. Skip processing if the strand is not +, -, or blank
  12. Report information to curators, but load anyway:
    1. if the chromosome in the input file does not match the chromosome for a marker
    2. list the new 'source/display' pairs not already in database
    3. display the Build text in the file
